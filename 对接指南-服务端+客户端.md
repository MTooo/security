# 对接指南 — 服务端 + 客户端

> 适用场景：你的应用既对外暴露加密接口（服务端），也需要主动调用其他加密服务（客户端）。

---

## 对接 Checklist

双方上线前逐项确认：

- [ ] 双方已生成密钥并交换公钥
- [ ] 客户端 peers 中 server-id 与服务端 server-id 一致
- [ ] 服务端 peers 中配了客户端的公钥和 server-id
- [ ] 网络互通（服务端 URL 可达）
- [ ] 测试握手成功（看日志，确认无异常码）
- [ ] 生产环境 `include-error-detail: false`
- [ ] 生产环境 `handshake-rate-limit-per-second` 已按需调整
- [ ] 纯客户端应用已设 `server-role: false`（减小攻击面）

## 前置条件

- **Spring Boot 3.x**：JDK 17+、Maven 3.6+，引入 `sm2-sdk-spring-boot3-starter`
- **Spring Boot 2.7**：JDK 8/11、Maven 3.6+，引入 `sm2-sdk-spring-boot-starter`
- 从 SDK 提供方获取对应的 Starter JAR
- 你和对端双方各自生成 SM2 密钥对，并**交换公钥**

---

## 接入流程图

```
① 获取 JAR → pom.xml 加依赖
② 生成 SM2 密钥对
③ 配置 application.yml
④ 编写业务代码
⑤ 启动验证
```

---

## 第一步：引入依赖

安装 SDK 到本地 Maven 仓库，然后在 pom.xml 中添加标准 Maven 依赖：

```bash
# 在 sm2-sdk 项目根目录执行
cd sm2-sdk
mvn clean install -DskipTests
```

然后在你的项目 `pom.xml` 中添加：

```xml
<!-- Spring Boot 3.x -->
<dependency>
    <groupId>io.github.mtooo</groupId>
    <artifactId>sm2-sdk-spring-boot3-starter</artifactId>
    <version>1.0.0</version>
</dependency>

<!-- Spring Boot 2.7 -->
<dependency>
    <groupId>io.github.mtooo</groupId>
    <artifactId>sm2-sdk-spring-boot-starter</artifactId>
    <version>1.0.0</version>
</dependency>
```

> Maven 会自动引入全部传递依赖（bcprov、hutool-crypto/http/bloomFilter/json、caffeine），无需手动声明。

## 第二步：生成密钥

```bash
# 在已引入 SDK 依赖的项目中执行
mvn exec:java -Dexec.mainClass="io.github.mtooo.core.util.Sm2KeyGen"

# 或批量生成 3 对密钥
mvn exec:java -Dexec.mainClass="io.github.mtooo.core.util.Sm2KeyGen" -Dexec.args="3"
```

输出示例：

```

```

**私钥自己保留，公钥发送给对端。**

## 第三步：配置 application.yml

```yaml
sm2:
  sdk:
    # ============ 密钥（必填） ============
    sm2-private-key: "6adef3c8e509f314cd68b23c80c466376d3fc47a870d89655fcbfe3b8e5a5bb7"  # 自己生成的私钥
    sm2-public-key:  "04666a3314b15e9671834a70a2e729eda64283ab9a92d3fd308f469e8e1c4e760c2f5e97fa4dd41fa9bbf0c0ae8d5b2db2e900fdd8cebd4f9de2436f25384006ce"  # 自己生成的公钥（给对方）

    server-id: my-server          # 我作为服务端时的身份标识（对方 peers 中配置）

    # ============ 分布式（可选） ============
    redis-session-store: true     # 启用 Redis 存储会话密钥（多实例必开）
    redis-key-prefix: sm2-server  # Redis 键前缀
    local-secret-key: "vsbQ8r4CGfpUYK8MwojXUw=="  # SM4 密钥（Base64），对 Redis 中的密钥做二次加密，防止脱库

    # ============ 安全（可选，有默认值，以下均为默认值） ============
    server-role: true                     # 服务端角色：true=注册握手端点和拦截器（默认），false=纯客户端
    handshake-rate-limit-per-second: 5    # 每秒最大握手次数，超过返回 408，防止 CPU/内存耗尽
    timestamp-window-ms: 30000            # 握手时间戳有效期(ms)，超时拒绝，防止重放攻击
    max-request-body-size: 1048576        # 请求体最大字节数（默认 1MB），超过拒绝，防止大密文 OOM
    include-error-detail: false           # 生产环境必须 false，防止泄露内部实现细节

    # ============ 对端配置 ============
    peers:
      # 对端 A（你的 peer 服务）
      - public-key: "04aaabbb..."         # 对端 A 的公钥
        server-url: "https://service-a.com"  # 对端 A 的服务地址
        server-id: peer-server            # 对端 A 的 server-id（须与对方配置一致）

      # 本机闭环测试（可选，用于本地验证）
      - public-key: "04ecc36a..."         # 自己的公钥
        server-url: "http://localhost:8080"
        server-id: my-server              # 与全局 server-id 一致
```

## 第四步：编写业务代码

### 作为服务端 — 接收加密请求

在 Controller **类或方法**上标注 `@Sm2Secured`，SDK 自动完成请求解密和响应加密：

```java
import io.github.mtooo.core.annotation.Sm2Secured;

@Sm2Secured
@RestController
public class MyController {

    @PostMapping("/api/echo")
    public Map echo(@RequestBody Map body) {
        body.put("serverTime", System.currentTimeMillis());
        return body;  // 请求自动解密，响应自动加密
    }
}
```

### 作为客户端 — 发起加密调用

注入 `Sm2HttpClient`，SDK 内部自动完成握手、加密请求、解密响应：

```java
import io.github.mtooo.client.Sm2HttpClient;

@RestController
public class ClientController {

    @Autowired
    private Sm2HttpClient sm2Client;

    public void callPeer() {
        // GET 请求
        UserInfo user = sm2Client.get("/api/user/query")
                .param("name", "张三")
                .execute(UserInfo.class);

        // POST 请求
        Result r = sm2Client.post("/api/payment/submit")
                .body(orderRequest)
                .execute(Result.class);
    }
}
```

> 未标注 `@Sm2Secured` 的端点完全不受影响，可混合使用。

### 客户端访问控制（可选）

限制不同客户端只能访问授权给它的路径：

```yaml
sm2:
  sdk:
    client-access:
      enabled: true           # 启用访问控制（默认 false）
      default-policy: deny    # 无匹配规则时的默认策略：allow / deny
      rules:
        - client-id: "app-a"
          paths:
            - /api/user/**
            - /api/order/**
        - client-id: "app-b"
          paths:
            - "/**"           # app-b 可访问一切
```

## 第五步：启动验证

```bash
# 1. 自测：获取会话（自动完成自我握手）
curl http://localhost:8080/api/session
# → {"sessionId": "uuid-xxx"}

# 2. 加密测试
curl -X POST http://localhost:8080/api/encrypt \
  -H "X-Session-Id: <sessionId>" \
  -H "Content-Type: text/plain" \
  -d '{"name":"test"}'
# → {"ciphertext": "base64密文..."}
```

启动后观察日志，确认握手成功无异常。然后与对端联调，验证双向加密通信。

## 配置参考表

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `sm2.sdk.enabled` | `true` | SDK 开关 |
| `sm2.sdk.sm2-private-key` | — | 本方 SM2 私钥（64 位十六进制） |
| `sm2.sdk.sm2-public-key` | — | 本方 SM2 公钥（130 位十六进制） |
| `sm2.sdk.server-url` | — | 对方服务地址（主动调用时必配） |
| `sm2.sdk.client-id` | `"default"` | 客户端标识，握手时发送给服务端，用于服务端访问控制 |
| `sm2.sdk.server-id` | `"default"` | 服务端标识，用于 SM2 握手 ZB 计算。对端 `peers[].server-id` 须与此一致 |
| `sm2.sdk.peers` | — | 对端配置列表 |
| `sm2.sdk.peers[].public-key` | — | 对端 SM2 公钥（130 位十六进制） |
| `sm2.sdk.peers[].server-url` | — | 对端服务地址 |
| `sm2.sdk.peers[].server-id` | `"default"` | 对端服务端标识，须与对方 `server-id` 一致 |
| `sm2.sdk.session-timeout-ms` | `300000` | 会话空闲超时 (ms) |
| `sm2.sdk.max-session-lifetime-ms` | `3600000` | 会话最大生命周期 (ms) |
| `sm2.sdk.max-session-requests` | `1000` | 单会话最大请求数 |
| `sm2.sdk.handshake-timeout-ms` | `10000` | 握手超时 (ms) |
| `sm2.sdk.max-sessions` | `10000` | 最大并发会话数 |
| `sm2.sdk.session-cleanup-interval-ms` | `60000` | 会话清理间隔 (ms) |
| `sm2.sdk.local-secret-key` | — | SM4 密钥（Base64），加密 Redis 中的 SM4 临时密钥 |
| `sm2.sdk.redis-key-prefix` | `sm2` | Redis 键前缀 |
| `sm2.sdk.redis-session-store` | `false` | 启用 Redis 会话存储 |
| `sm2.sdk.client-access.enabled` | `false` | 启用客户端访问控制 |
| `sm2.sdk.client-access.default-policy` | `allow` | 默认策略：`allow` / `deny` |
| `sm2.sdk.client-access.rules[].client-id` | — | 客户端标识 |
| `sm2.sdk.client-access.rules[].paths` | — | 允许的路径模式列表（Ant 风格） |
| `sm2.sdk.server-role` | `true` | 服务端角色开关。`false` 时不注册握手端点和拦截器，仅保留客户端能力 |
| `sm2.sdk.handshake-rate-limit-per-second` | `10` | 握手速率限制（每秒最大请求数）。公网建议 5~10，内网可放宽 |
| `sm2.sdk.timestamp-window-ms` | `30000` | 握手时间戳有效窗口（毫秒）。超时拒绝，防止重放攻击 |
| `sm2.sdk.max-request-body-size` | `1048576` | 最大请求体大小（字节，默认 1MB）。超过拒绝，防止大密文 OOM |
| `sm2.sdk.include-error-detail` | `false` | 错误详情开关。生产环境必须 `false`，仅调试环境临时开启 |

## 异常码速查

| 错误码 | HTTP 状态码 | 含义 | 处理建议 |
|--------|-------------|------|----------|
| `11301` | 401 | 会话已过期 | SDK 自动重握手 |
| `11302` | 401 | 请求次数超限 | SDK 自动重握手 |
| `22301` | 401 | 服务端会话不存在 | SDK 自动重握手 |
| `21202` | 400 | TAG 校验失败 | 废弃会话，自动重握手 |
| `21103` | 400 | 服务端证书验签失败 | 检查对方公钥配置是否正确 |
| `11108` | 408 | 握手超时 | 检查网络连通性 |
| `19003` | 503 | 握手熔断 | 30 秒后自动恢复 |
| `29001` | 403 | Nonce 重放 | 安全告警，检查是否有重放攻击 |
| `22502` | 403 | 客户端无权访问该路径 | 检查 client-access 配置或 clientId |
| `31401` | 500 | 私钥未配置 | 检查 yml 中 sm2-private-key |
