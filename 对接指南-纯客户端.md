# 对接指南 — 纯客户端角色

本文档面向**只需要作为客户端调用加密 API** 的用户。场景：你的应用调用另一个服务端的 SM2 加密接口，自己不需要暴露握手端点。

---

## 对接 checklist

- [ ] 已从 SDK 提供方获取 JAR 并添加到 `lib/` 目录
- [ ] 已生成自己的 SM2 密钥对
- [ ] 公钥已发给服务端（对方确认已配置到 peers 中）
- [ ] `client-id` 与服务端 `peers[].server-id` 一致
- [ ] 服务端的公钥和 URL 配置正确
- [ ] `server-role: false`（纯客户端模式，不暴露服务端端点）
- [ ] 项目启动无异常
- [ ] 测试调用成功后返回预期的业务数据



## 前置条件

- **Spring Boot 3.x**：JDK 17+、Maven 3.6+，引入 `sm2-sdk-spring-boot3-starter`
- **Spring Boot 2.7**：JDK 8/11、Maven 3.6+，引入 `sm2-sdk-spring-boot-starter`
- 从 SDK 提供方获取对应的 Starter JAR
- 从你要调用的服务端获取其 **SM2 公钥**

---

## 接入流程图

```
① 获取 JAR → pom.xml 加依赖
② 生成 SM2 密钥对
③ 把公钥发给服务端（让对方配到 peers 里）
④ 配置 application.yml
⑤ 编写调用代码
⑥ 启动验证
```

---

## 第一步：引入依赖

安装 SDK 到本地 Maven 仓库，然后在 pom.xml 中添加标准 Maven 依赖：

```bash
# 在 sm2-sdk 项目根目录执行
cd sm2-sdk
mvn clean install -DskipTests
```

然后在你的项目 `pom.xml` 中添加：

```xml
<!-- Spring Boot 3.x -->
<dependency>
    <groupId>io.github.mtooo</groupId>
    <artifactId>sm2-sdk-spring-boot3-starter</artifactId>
    <version>1.0.0</version>
</dependency>

<!-- Spring Boot 2.7 -->
<dependency>
    <groupId>io.github.mtooo</groupId>
    <artifactId>sm2-sdk-spring-boot-starter</artifactId>
    <version>1.0.0</version>
</dependency>
```

> Maven 会自动引入全部传递依赖（bcprov、hutool-crypto/http/bloomFilter/json、caffeine），无需手动声明。

---

## 第二步：生成密钥

```bash
# 在已引入 SDK 依赖的项目中执行
mvn exec:java -Dexec.mainClass="io.github.mtooo.core.util.Sm2KeyGen"

# 或批量生成 3 对密钥
mvn exec:java -Dexec.mainClass="io.github.mtooo.core.util.Sm2KeyGen" -Dexec.args="3"
```

输出示例：

```

sm2-public-key:  04666a3314b15e9671834a70a2e729eda64283ab9a92d3fd308f469e8e1c4e760c2f5e97fa4dd41fa9bbf0c0ae8d5b2db2e900fdd8cebd4f9de2436f25384006ce
sm4-key:         bec6d0f2be0219fa5460af0cc288d753  (hex, 可用作 local-secret-key)
sm4-key-base64:  vsbQ8r4CGfpUYK8MwojXUw==  (base64, 可直接填入 local-secret-key)

```

**私钥自己保留，公钥发给服务端。**

---

## 第三步：把公钥发给服务端

将上一步生成的 `sm2-public-key` 发给服务端团队。服务端需要把它配到自己的 `peers` 列表中，并关联你的 `client-id`：

```yaml
# 服务端侧的配置（你不需要配这个，告知对方即可）
sm2:
  sdk:
    peers:
      - public-key: "<你的公钥>"    # ← 你生成并发给对方的
        client-id: "<你的 client-id>"  # ← 你在第四步配的 client-id
```

> 务必和服务端确认：对方已配置你的公钥，且双方 `client-id`/`server-id` 一致。

---

## 第四步：配置 application.yml

```yaml
sm2:
  sdk:
    # ===== 密钥（必填） =====
    sm2-private-key: "6adef3c8e509f314cd68b23c80c466376d3fc47a870d89655fcbfe3b8e5a5bb7"     # 自己的私钥
    sm2-public-key:  "04666a3314b15e9671834a70a2e729eda64283ab9a92d3fd308f469e8e1c4e760c2f5e97fa4dd41fa9bbf0c0ae8d5b2db2e900fdd8cebd4f9de2436f25384006ce"     # 自己的公钥

    # ===== 身份（必填） =====
    client-id: my-client               # 告诉服务端"我是谁"

    # ===== 分布式部署（可选） =====
    redis-session-store: true           # 启用 Redis 存储会话密钥（多实例必开）
    redis-key-prefix: sm2-client        # Redis 键前缀，与对端区分避免冲突
    local-secret-key: "vsbQ8r4CGfpUYK8MwojXUw=="  # 对 Redis 中的密钥做 SM4 二次加密，防止脱库

    # ===== 纯客户端模式（关闭服务端端点，减小攻击面） =====
    server-role: false                  # 不注册 /handshake 端点，只主动调用

    # ===== 对端配置 =====
    peers:
      - public-key: "04ecc36a..."      # 服务端的公钥
        server-url: "https://api.example.com"  # 服务端地址
        server-id: their-server         # 必须 = 服务端的 server-id
```

> **`client-id` 必须与对方 `peers[].server-id` 对齐**：你在配置中声明的 `client-id`，服务端需要在它的 peers 列表中用相同的值来关联你的公钥。

---

## 第五步：编写调用代码

```java
@RestController
public class MyClientController {

    @Autowired
    private Sm2HttpClient sm2Client;

    @GetMapping("/call-echo")
    public Map callEcho() {
        // 自动握手 → 加密 → 发送 → 解密响应 → 返回明文
        return sm2Client.post("/api/echo")
                .body(Map.of("msg", "hello"))
                .execute(Map.class);
    }

    @GetMapping("/query-user")
    public UserInfo queryUser() {
        return sm2Client.get("/api/user/query")
                .param("name", "张三")
                .execute(UserInfo.class);
    }
}
```

`Sm2HttpClient` 内部自动完成：**握手 → 加密请求 → 发送 → 解密响应 → 返回明文**。会话过期自动重握手，对调用方完全透明。支持 GET / POST / PUT / DELETE 四种方法。

---

## 第六步：启动验证

```bash
curl http://localhost:8080/call-echo
# → 自动握手 → 加密请求 → 解密响应 → 返回明文结果
```

如果返回了预期的业务数据，说明对接成功。如果返回 401 或握手失败，请检查：
- 密钥是否配对（你的私钥 vs 你的公钥，服务端的公钥是否正确）
- `client-id` 是否与服务端配置一致
- 服务端地址是否可达

## 配置参考表

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `sm2.sdk.sm2-private-key` | — | 本方 SM2 私钥（64 位十六进制） |
| `sm2.sdk.sm2-public-key` | — | 本方 SM2 公钥（130 位十六进制） |
| `sm2.sdk.client-id` | `"default"` | 客户端标识，握手时发送给服务端 |
| `sm2.sdk.peers` | — | 对端配置列表 |
| `sm2.sdk.peers[].public-key` | — | 对端 SM2 公钥 |
| `sm2.sdk.peers[].server-url` | — | 对端服务地址 |
| `sm2.sdk.peers[].server-id` | `"default"` | 对端服务端标识，须与对方 `server-id` 一致 |
| `sm2.sdk.server-role` | `true` | 服务端角色开关。纯客户端设为 `false` |
| `sm2.sdk.redis-session-store` | `false` | 启用 Redis 会话存储 |
| `sm2.sdk.redis-key-prefix` | `sm2` | Redis 键前缀 |
| `sm2.sdk.local-secret-key` | — | SM4 密钥（Base64），加密 Redis 中的密钥 |
| `sm2.sdk.session-timeout-ms` | `300000` | 会话空闲超时 (ms) |
| `sm2.sdk.max-session-lifetime-ms` | `3600000` | 会话最大生命周期 (ms) |
| `sm2.sdk.max-session-requests` | `1000` | 单会话最大请求数 |
| `sm2.sdk.handshake-timeout-ms` | `10000` | 握手超时 (ms) |
| `sm2.sdk.include-error-detail` | `false` | 错误详情开关，生产必须 `false` |

## 异常码速查

| 错误码 | HTTP | 含义 | 处理方式 |
|--------|------|------|---------|
| `11301` | 401 | 会话已过期 | SDK 自动重握手 |
| `11302` | 401 | 请求次数超限 | SDK 自动重握手 |
| `22301` | 401 | 服务端会话不存在 | SDK 自动重握手 |
| `21202` | 400 | TAG 校验失败 | 废弃会话 + 重握手 |
| `21103` | 400 | 服务端证书验签失败 | 检查对方公钥配置是否正确 |
| `29001` | 403 | Nonce 重放 | 安全告警，联系服务端 |
| `22502` | 403 | 客户端无权访问该路径 | 检查 clientId 是否与服务端配置一致 |
| `11108` | 408 | 握手超时 | 检查网络连通性 |
| `19003` | 503 | 握手熔断 | 30s 后自动恢复，检查网络或握手频率 |
| `31401` | 500 | 私钥未配置 | 检查 `sm2.sdk.sm2-private-key` 是否填写 |
